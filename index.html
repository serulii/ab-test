A/B Testing

CS1300

Overview

In this scenario, I designed for a website (MedX) that allows users to schedule doctor's appointments. I wanted to see if making a small change to the UI of the website--primarily with spacing and color choices--would make it more convenient and intuitive for users to navigate the page and book the appointment they wanted.

In other words: will changing the UI of MedX help users find and book an appointment faster?

I used an A/B test to help answer this question. By changing the website slightly and tracking how quickly and easily users were able to navigate to the correct appointment and click the "schedule appointment" button on both old and new versions of the website, I could analyze how a small change in the UI affected the user's experience.

Design changes

The changes I made include:

  • Separating appointments into distinct boxes
  • Increasing spacing between appointments
  • Changing the color of the appointment button to green
  • Increasing contrast of text on buttons

Metrics & Analysis

The metrics I chose to analyze were:

  1. Misclick rate (the frequency with which users click something else on the page before finding the correct button for the task)
  2. Time on page (time spent on the webpage for each user group)
  3. Mouse move distance (the amount they moved the mouse)

For each metric, I created the following hypotheses.

MISCLICK RATE

  • Null hypothesis: The frequency with which users click something other than the correct button is about the same between version A and version B.
  • Alternative hypothesis: The frequency with which users click something other than the correct button is different between version A and version B.
  • Reasoning: Since version B has the "schedule appointment" buttons highlighted in green and spacing to separate different appointments, users will be less likely to misclick on parts of the page that aren't related to their goal or on the wrong appointment.
  • Prediction: I predict that we will be able to reject the null hypothesis since version A has a design that makes it easy to misclick. The buttons are spaced closely together with no separation between separate appointments.

TIME ON PAGE

  • Null hypothesis: The total amount of time spent on the webpage is about the same between version A and version B.
  • Alternative hypothesis: The total amount of time spent on the webpage is greater in version A compared to version B.
  • Reasoning: If version B has a more intuitive design overall, then users will spend less time searching for the button they want & thus spend less time overall on the page.
  • Prediction: I predict that we will be able to reject the null hypothesis. However, part of this may be since users experienced version A first and thus already had some experience with the interface.

MOUSE MOVE DISTANCE

  • Null hypothesis: The amount users move the mouse is about the same between version A and version B.
  • Alternative hypothesis: The amount users move the mouse is greater in version A compared to version B.
  • Reasoning: This makes the assumption that users who are confused or looking at more parts of the page will move the mouse more to look for the correct button. If version B has a clearer design, then users will move the mouse less.
  • Prediction: I predict that we may fail to reject the null hypothesis because users move the mouse differently. Some users may prefer to observe a website before moving the mouse.

Statistical Analysis

I then performed statistical tests on each metric to determine if verison A and version B had significant differences in user experience.

MISCLICK RATE

Since miclick rate is categorical data (either true or false), I chose a chi-square test with a significance level of 0.05 to determine if there was a significant difference between page A and B.

The resulting p-value was 0.0032, a lot lower than our chosen significance level. This means that there is only a 0.32% chance that there is no difference between the two groups.

As a result, we can conclude that there is significant evidence that the alternative hypothesis is true, and we can reject the null hypothesis. In other words, there is a difference in the misclick rate on version A and version B of the website.

TIME ON PAGE

I chose to run a one-tailed t-test because we are trying to find if users spent more time on page A compared to B. Again, I chose a significance level of 0.05.

The average time users spent on version A is 29411.25ms, while the average time users spent in version B is 10479.27ms. Overall, users spent more time on version A on average.

The p-value (that A>B) turned out to be 0.000566. This is much lower than our significance level of 0.05, meaning that the difference between A and B with respect to the time users spent on the page is statistically significant!

As a result, we reject our null hypothesis. There is thus statistically significant evidence that the alternative hypothesis is true: users spend more time on website A compared to website B.

MOUSE MOVE DISTANCE

Similarly, I used a one-tailed t-test here--we're trying to find if users moved the mouse more in version A compared to B.

The resulting p-value was 0.00172, which is (surprisingly!) also much lower than our significance level of 0.05.

Here, we result our null hypothesis as well and can conclude that there is significant evidence that the alternative hypothesis, users moved the mouse more on website A compared to B, is true!

Looking at the averages, users moved the mouse on average 4793.82 pixels more in version A compared to version B. This further supports our alternative hypothesis.

Conclusion

Based on the analysis above, there is a significant difference between the misclick rate on versions A and B of the website, and users overall spent more time and moved the mouse more on version A compared to B. Since our goal is to make scheduling an appointment as fast and clear as possible, Version B of the website offers an improvement in user experience over version A.
